🏗️ Architecture

This vocabulary implements a "small waist" architecture where the OWL ontology and SHACL shapes serve as the single source of semantic truth:

┌─────────────────────────┐    ┌─────────────────────────┐
│ OWL Ontology            │    │ SHACL Shapes            │
│ (What CAN exist)        │    │ (What MUST exist)       │
└─────────┬───────────────┘    └─────────┬───────────────┘
          │                              │
          └─────────────┬──────────────────┘
                        │
            ┌───────────▼────────────┐
            │ JSON Schema Generator  │
            └───────────┬────────────┘
                        │
     ┌──────────────────┼──────────────────┐
     │                  │                  │
┌────▼────┐   ┌────────▼────────┐   ┌─────▼─────┐
│ Web APIs│   │ Database Schema │   │ UI Forms  │
│ OpenAPI │   │ SQL DDL         │   │ Validation│
└─────────┘   └─────────────────┘   └───────────┘

Key Features:

  • Schema.org Compatibility - Extends standard Action class
  • Hierarchical Structure - 6-level depth organization (Root → Child → Leaf)
  • GTD Integration - Context tags and project organization
  • Temporal Model - Do-date scheduling + due-date deadlines
  • iCalendar Recurrence - RRULE pattern compatibility
  • State Management - Extended beyond simple completion tracking

🔗 Content Negotiation

The same URLs serve different formats based on your Accept headers:

Accept: text/turtle → actions/vocabulary.ttl
Accept: application/json → actions/schemas/actions-combined.schema.json
Accept: text/html → Human-readable documentation
