The Actions Vocabulary implements a hierarchical task management system with the following key features:

Class Hierarchy

Action (base class)
‚îú‚îÄ‚îÄ RootAction (depth 0) - Top-level actions with project organization
‚îú‚îÄ‚îÄ ChildAction (depth 1-4) - Nested actions with parent relationships  
‚îî‚îÄ‚îÄ LeafAction (depth 5) - Deepest level actions, cannot have children

Key Properties

  • Identification: uuid (Version 7 UUID)
  • Hierarchy: parentAction, depth
  • Organization: project (root actions only)
  • Context: context (GTD-style @contexts)
  • Priority: priority (1-4 Eisenhower Matrix)
  • Timing: doDate/doTime, dueDate/dueTime
  • Recurrence: iCalendar RRULE compatible properties
  • State: state (active, completed, cancelled, etc.)
